# Gemini Image Style Guide
Project: AI-Powered Grant Evaluation Accelerator (GrantScoreCard)

Purpose: Ensure all generated visuals for the repository (headers, diagrams, icons, README banners) follow a consistent design language suitable for developer documentation and Salesforce ecosystem projects.

## Core Visual Style
Use a modern SaaS product illustration style similar to:
- Stripe developer documentation
- Linear app illustrations
- Salesforce ecosystem visuals
- modern GitHub README product banners

The overall feel should be:
- clean
- minimal
- professional
- developer-friendly
- modern SaaS platform

Avoid:
- cartoon styles
- clip art
- heavy gradients
- excessive icons
- busy backgrounds
- sci-fi aesthetics

## Color Palette
Primary palette:
- Deep Indigo / Navy: `#1E2A4A`
- Salesforce Blue: `#0176D3`
- Soft Purple Accent: `#7B8CFF`
- Light Background: `#F7F9FC`
- Soft Gray: `#E6EAF0`

Gradients (use lightly and sparingly):
- Indigo -> Purple
- Blue -> Violet

Avoid:
- neon colors
- orange/red dominant palettes
- dark backgrounds unless the image represents a developer or terminal theme

## Typography Style
If text appears inside images:

Headings:
- Bold
- Clean sans-serif
- Large
- Highly readable

Example heading:
- AI-Powered Grant Evaluation Accelerator

Subtitles:
- lighter weight
- professional
- simple

Example subtitle:
- Design, evaluate, and continuously improve grant scoring frameworks with AI.

Avoid decorative or stylized fonts.

## Illustration Style
Use soft layered UI cards or tiles that resemble modern product dashboards.

Preferred visual elements:
- floating platform tiles
- evaluation scorecards
- document forms
- review checklists
- analytics charts
- evaluation frameworks
- criteria libraries
- application review dashboards
- AI analysis panels

Connections between elements:
- thin connector lines
- minimal arrows
- clean workflow progression

Everything should look like a clean SaaS workflow diagram.

## Layout Rules
Preferred layout orientation:
- horizontal compositions for README banners
- structured workflow layouts for diagrams

Recommended image sizes:
- Hero banner: `1600 x 400`
- Section headers: `1600 x 200`
- Architecture diagrams: `1600 x 900`
- Icons: square format

Spacing:
- large whitespace
- balanced margins
- no clutter

Visual flow direction:
- left -> right
- or
- top -> bottom

## Workflow Visualization Style
When illustrating the system architecture or process flow, use simple labeled tiles connected by arrows.

Example flow structure:
- Funding Opportunity
- Grant Application Intake
- Evaluation Framework Design
- Reviewer Scorecards
- AI Framework Review
- Criteria Library Governance
- Funding Decision Insights

Each step should appear as a minimal UI card or floating tile.

## AI Visualization Style
Represent AI components subtly using:
- glowing nodes
- AI chips
- analysis panels
- recommendation cards
- network connection points

Avoid:
- robots
- humanoid AI
- sci-fi imagery

The AI should feel like intelligent software, not robotics.

## Developer / Debug Theme Variant
For developer-focused sections such as debugging or logging:

Background:
- dark terminal panel

Elements:
- terminal windows
- structured JSON blocks
- code brackets
- flow logs
- system arrows

Even in debug visuals maintain the same blue/purple accent palette.

## Icon Style
Icons should be:
- minimal
- flat
- consistent line weight
- soft filled or outline style

Common icon subjects:
- documents
- checklists
- databases
- AI nodes
- charts
- workflow connections

Avoid emoji-style or cartoon icons.

## Consistency Rule
All generated visuals must appear to belong to a single product design system.

Common visual traits:
- floating UI cards
- soft shadows
- thin connector arrows
- white or light backgrounds
- blue and purple palette
- modern SaaS platform aesthetic

## Standard Gemini Prompt Prefix
Use this prefix before any Gemini image request:

> Use the AI-Powered Grant Evaluation Accelerator visual style guide.
>
> Modern SaaS product illustration, minimal and professional, Salesforce-inspired blue/indigo/purple palette, clean white background, floating UI tiles with soft shadows, structured workflow arrows, and developer documentation quality graphics suitable for a GitHub README.

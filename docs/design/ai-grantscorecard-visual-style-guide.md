# AI-Powered Grant Evaluation Accelerator Visual Style Guide
Project: AI-Powered Grant Evaluation Accelerator (GrantScoreCard)

Purpose: Ensure all generated visuals for the repository (headers, diagrams, icons, README banners, and UI mockups) follow one consistent design system suitable for developer documentation and Salesforce ecosystem projects.

## Core Visual Style
Use a modern SaaS product illustration style inspired by:
- Stripe developer documentation
- Linear app illustrations
- Salesforce ecosystem visuals
- modern GitHub README product banners

Overall feel:
- clean
- minimal
- professional
- developer-friendly
- modern SaaS platform

Avoid:
- cartoon styles
- clip art
- heavy gradients
- excessive icons
- busy backgrounds
- sci-fi aesthetics

## Color Palette
Primary palette:
- Deep Indigo / Navy: `#1E2A4A`
- Salesforce Blue: `#0176D3`
- Soft Purple Accent: `#7B8CFF`
- Light Background: `#F7F9FC`
- Soft Gray: `#E6EAF0`

Gradient guidance (use lightly):
- Indigo -> Purple
- Blue -> Violet

Avoid:
- neon colors
- orange/red dominant palettes
- dark backgrounds except deliberate developer/debug themes

## Typography Style
When text appears in images:
- Heading: bold, clean sans-serif, large, highly readable
- Subtitle: lighter weight, simple, professional

Example heading:
- AI-Powered Grant Evaluation Accelerator

Example subtitle:
- Design, evaluate, and continuously improve grant scoring frameworks with AI.

Avoid decorative or stylized fonts.

## Illustration Style
Use soft layered UI cards/tiles resembling modern product dashboards.

Preferred elements:
- floating platform tiles
- evaluation scorecards
- document forms
- review checklists
- analytics charts
- evaluation frameworks
- criteria libraries
- application review dashboards
- AI analysis panels

Connection style:
- thin connector lines
- minimal arrows
- clean workflow progression

## Layout Rules
Preferred orientation:
- horizontal compositions for README banners
- structured workflow layouts for architecture diagrams

Standard image sizes:
- Hero banner: `1600 x 400`
- Section header: `1600 x 200`
- Architecture diagram: `1600 x 900`
- Icon: square format

Spacing rules:
- large whitespace
- balanced margins
- no clutter

Flow direction:
- left -> right
- or top -> bottom

## Workflow Visualization Standards
For process diagrams, use simple labeled tiles connected by subtle arrows.

Canonical flow example:
- Funding Opportunity
- Grant Application Intake
- Evaluation Framework Design
- Reviewer Scorecards
- AI Framework Review
- Criteria Library Governance
- Funding Decision Insights

Each step should appear as a minimal UI card or floating tile.

## AI Visualization Rules
Represent AI features as intelligent software components, not robots.

Use:
- glowing nodes
- AI chips
- analysis panels
- recommendation cards
- network connection points

Avoid:
- humanoid AI
- robots
- sci-fi motifs

## Developer / Debug Variant
For debug/log visuals:
- dark terminal panel background
- terminal windows
- structured JSON blocks
- code brackets
- flow logs
- system arrows

Keep blue/indigo/purple accents consistent even in debug mode.

## Icon Style Rules
Icons must be:
- minimal
- flat
- consistent line weight
- soft filled or outline style

Common icon subjects:
- documents
- checklists
- databases
- AI nodes
- charts
- workflow connections

Avoid emoji-style or cartoon icons.

## Consistency Rule
All visuals must look like one unified product design system.

Shared traits:
- floating UI cards
- soft shadows
- thin connector arrows
- white or light backgrounds
- blue and purple palette
- modern SaaS platform aesthetic

## Standard Gemini Prompt Prefix
Use this prefix for every Gemini image request:

> Use the AI-Powered Grant Evaluation Accelerator visual style guide.
>
> Modern SaaS product illustration, minimal and professional, Salesforce-inspired blue/indigo/purple palette, clean white background, floating UI tiles with soft shadows, structured workflow arrows, and developer documentation quality graphics suitable for a GitHub README.
